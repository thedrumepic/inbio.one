# Исправление ошибки "Not Found" на главной странице

Проблема: Бэкенд не мог найти файл `index.html`, потому что мы неправильно настроили передачу файлов между контейнерами фронтенда и бэкенда.

## Что исправлено
1.  **docker-compose.yml**:
    *   Фронтенд теперь сохраняет файлы из правильной папки `/usr/share/nginx/html` в общий том.
    *   Бэкенд получает эти файлы в папку `/frontend/build`, где он их и ожидает увидеть.

## Как применить (Обязательно!)

Выполните эти команды на сервере:

1.  **Остановите и удалите старые контейнеры и тома**:
    (Важно удалить том, чтобы он пересоздался с правильными файлами)
    ```bash
    docker-compose down -v
    ```
    *Внимание: `down -v` удалит и данные базы. Если база важна, используйте просто `docker-compose down` и вручную удалите том `inbio_frontend_build` (название может отличаться, проверьте через `docker volume ls`)*.
    
    **Безопасный вариант (если база важная):**
    ```bash
    docker-compose down
    docker volume rm inbio-v2_frontend_build
    # (или code_frontend_build - посмотрите точное имя через docker volume ls)
    docker-compose up -d --build
    ```

2.  **Простая перезагрузка (если база пустая или тестовая)**:
    ```bash
    docker-compose down -v && docker-compose up -d --build
    ```

Теперь главная страница должна открываться корректно, а заголовок вкладки будет "InBio.one" (или ваш настроенный).
