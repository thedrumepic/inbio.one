# Руководство по исправлению визуальных багов деплоя

Если после деплоя вы видите `__SEO_TITLE__` в заголовке или битый логотип, выполните следующие действия.

## 1. Что мы исправили в коде
Мы внесли 3 критических изменения, чтобы проект работал в Docker "из коробки":
1.  **Относительные пути на фронтенде**: Теперь фронтенд не ищет бэкенд на `localhost:8000`, а использует относительные пути. Это позволяет Caddy правильно перехватывать запросы.
2.  **SEO для главной страницы**: В `server.py` добавлен маршрут для `/`, чтобы главная страница тоже получала красивые заголовки и описания.
3.  **Умный роутинг в Caddy**: Мы перенастроили `Caddyfile`, чтобы он отправлял все запросы к страницам (HTML) на бэкенд для обработки SEO, а картинки и скрипты брал напрямую с фронтенда.

## 2. Как применить изменения

### Шаг 1: Пересборка контейнеров
Так как мы изменили JS-код фронтенда, контейнеры нужно пересобрать:
```bash
docker-compose down
docker-compose up -d --build --remove-orphans
```

### Шаг 2: Проверка папок
Убедитесь, что на вашем сервере в папке с проектом есть логотип:
`./backend/uploads/logo/logo.png` и `logo-dark.png`.
Если папки `uploads` нет, создайте её:
```bash
mkdir -p backend/uploads/logo
# Скопируйте туда ваши логотипы
```

### Шаг 3: Очистка кэша браузера
Браузер мог запомнить старый `index.html`. Нажмите **Ctrl + F5**, чтобы принудительно обновить страницу.

## 3. Почему это произошло?
*   **SEO**: Caddy видел, что у фронтенда есть файл `index.html`, и отдавал его "как есть", не спрашивая бэкенд. Теперь бэкенд — главный по выдаче HTML.
*   **Лого**: В коде была заглушка `localhost:8000` на случай отсутствия переменной окружения. Теперь там пустота, что заставляет браузер искать файл на вашем домене `inbio.one`.
